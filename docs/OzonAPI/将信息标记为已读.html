将信息标记为已读

post
/v2/chat/read
  • 描述和范例
  • 控制台

一种将选定的信息和其之前的信息标记为已读的方法。

header Parameters
Client-Id
required
string

用户识别号。

Api-Key
required
string

API-密钥。

Request Body schema: application/json
chat_id
required
string

聊天识别码。

from_message_id
integer <uint64>

信息识别码。

回复

Response Schema: application/json
unread_count
integer <int64>

聊天中未读消息的数量。

请求范例

    Content type
    application/json
    {
    • "chat_id": "99feb3fc-a474-469f-95d5-268b470cc607",
    • "from_message_id": 3000000000118032000
    }

    回复范例

    • 200
    • 400
    • 403
    • 404
    • 409
    • 500
    Content type
    application/json
    {
    • "unread_count": 0
    }
