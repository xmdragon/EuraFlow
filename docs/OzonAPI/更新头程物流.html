post
/v1/warehouse/fbs/first-mile/update
  • 描述和范例
  • 控制台

您可以在 讨论 的评论中对此方法提供反馈 在 Ozon for dev 开发者社区中。

header Parameters
Client-Id
required
string

用户识别号。

Api-Key
required
string

API-密钥。

Request Body schema: application/json
drop_off_point_id
integer <int64>

订单发运点ID。如果first_mile_type = DROP_OFF，该参数必填。

first_mile_type
required
string
Default: "UNSPECIFIED"
Enum: "UNSPECIFIED" "PICK_UP" "DROP_OFF"

第一英里类型：

  • UNSPECIFIED — 未定义；
  • PICK_UP —发运给快递员；
  • DROP_OFF —发运至接收点。
warehouse_id
required
integer <int64>

仓库ID。

回复

Response Schema:
application/json
operation_id
string

操作ID。通过方法 /v1/warehouse/operation/status 获取操作状态。

请求范例

    Content type
    application/json
    {
    • "first_mile_type": "DROP_OFF",
    • "drop_off_point_id": 0,
    • "warehouse_id": 0
    }

    回复范例

    • 200
    • default
    Content type
    application/json
    {
    • "operation_id": "string"
    }
