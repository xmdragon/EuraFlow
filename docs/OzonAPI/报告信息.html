报告信息

post
/v1/report/info
  • 描述和范例
  • 控制台

通过识别码回送有关先前创建的报告的信息。

header Parameters
Client-Id
required
string

用户识别号。

Api-Key
required
string

API-密钥。

Request Body schema: application/json
code
required
string

报告的唯一识别码。

回复

Response Schema: application/json
object

关于报告的信息。

code
string

报告的唯一识别码。

created_at
string <date-time>

报告创建日期。

error
string

生成报告时的错误代码。

file
string

XLSX文件的链接。

SELLER_RETURNS 类型的报告，链接有效期为5分钟。

object

一个数组，包含卖家创建报告时指定的过滤器。

report_type
string

报告类型：

  • SELLER_PRODUCTS — 商品报告，
  • SELLER_TRANSACTIONS — 交易报告，
  • SELLER_PRODUCT_PRICES — 商品价格报告，
  • SELLER_STOCK — 商品库存报告，
  • SELLER_RETURNS — 退货报告，
  • SELLER_POSTINGS — 发货报告，
  • SELLER_FINANCE — 财务报告，
  • SELLER_PRODUCT_DISCOUNTED — 减价商品报告，
  • DOCUMENT_B2B_SALES — 面向法人客户的销售报告，
  • MUTUAL_SETTLEMENT — 结算报告，
  • COMPENSATION — 赔偿报告，
  • DECOMPENSATION — 赔偿返还报告。
status
string

报告生成状态：

  • waiting—在等待队列中待处理，
  • processing—正在处理，
  • success，
  • failed。
